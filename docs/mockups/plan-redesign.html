Fase 3: Rediseno de la Tabla de Contratos

Nuevas columnas con logica "+X", ordenacion, fila clickable, y cambios en el esquema de BD.

CAMBIO DE ESQUEMA FRONTEND BACKEND

Tareas

  • SCHEMA: Crear tabla intermedia ContractParcel (contractId, parcelId) para M:N contratos-parcelas
  • SCHEMA: Anadir campos al modelo Contract: nextStep (String?), nextStepDate (DateTime?), notes (String?)
  • SCHEMA: Anadir valor SIGNED_ADDENDUM al enum ContractStatus
  • MIGRATION: Migrar datos existentes (cada contract.parcelId -> ContractParcel)
  • BACKEND: Actualizar servicios y queries para la nueva relacion M:N
  • FRONTEND: Nuevas columnas: Municipio (+X), Parcela/s (+X), Sup. Catastral total, Propietario/s (+X), Estado, Next Step, Fecha Firma
  • FRONTEND: Eliminar columna "Precio" y boton "Ver"
  • FRONTEND: Hacer toda la fila clickable (redirige al detalle)
  • FRONTEND: Headers de columna ordenables (sort asc/desc)
  • FRONTEND: Boton "+ Anadir Contrato" que inserta fila vacia en modo edicion
  • FRONTEND: Filas con status EXPIRED fondo gris
  • FRONTEND: Badge "Firmado - Adenda" con icono documento
Cambios en Prisma Schema
// Nueva tabla intermedia
model ContractParcel {
  id         String   @id @default(cuid())
  contractId String
  contract   Contract @relation(...)
  parcelId   String
  parcel     Parcel   @relation(...)
  @@unique([contractId, parcelId])
}

// Nuevos campos en Contract
model Contract {
  // ... campos existentes ...
  nextStep     String?    // Texto libre "Next Step"
  nextStepDate DateTime?  // Fecha limite
  notes        String?    // Observaciones
  parcels      ContractParcel[]  // Nueva relacion M:N
}

// Nuevo valor en enum
enum ContractStatus {
  DRAFT
  ACTIVE
  EXPIRED
  SIGNED_ADDENDUM  // Firmado - Adenda
}
